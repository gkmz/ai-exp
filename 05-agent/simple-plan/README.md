# simple-plan

一个最简单可运行的计划 Agent 示例：通过自然语言描述今天的任务与约束，调用本地 Qwen 大模型生成带时间安排的执行计划。

## 你能学到什么

- 如何采集用户输入并组织为 Prompt
- 如何通过 OpenAI 兼容接口调用本地大模型
- 如何把模型输出约束成结构化任务计划

## 运行前准备

1. 本地启动 OpenAI 兼容服务（例如 Ollama）：
   - 示例：`ollama serve`
2. 本地准备一个 Qwen 模型：
   - 示例：`ollama pull qwen2.5:7b`
3. 环境变量（可选）：
   - `LLM_BASE_URL`：默认 `http://127.0.0.1:11434/v1`
   - `LLM_MODEL`：默认 `qwen2.5:7b`
   - `LLM_API_KEY`：默认 `local-dev-key`（本地多数场景可忽略）

## 快速开始

```bash
cd 05-agent/simple-plan
go mod tidy
go run .
```

程序会提示你用自然语言输入任务说明（可多行），输入空行后开始生成计划。

## 示例输入

```text
今天我有3个任务：
1）完成需求评审，比较重要，最好上午完成；
2）实现登录接口，这个比较紧急，17:00前必须提测；
3）编写接口测试，不那么紧急，但今天下班前想完成。
我下午14:00-15:00要开会，请帮我给出今天详细工作计划和时间安排。
```

## 说明

- 这是“最小实现”，重点是学习完整链路而不是复杂框架。
- 你可以下一步尝试：
  - 增加多轮对话（迭代优化计划）
  - 输出为 JSON 并保存为文件
  - 接入日历信息做自动避让（会议时段不排任务）
