# travel-planning

一个用于测试 Agent 规划能力的旅行规划 demo：工具数据纯本地模拟，但最终行程由本地大模型生成，形成完整闭环。

## 功能点

- 目的地固定为成都（按你的要求先做单目的地）
- 模拟多工具调用链路：天气工具 -> 酒店工具 -> 景点工具 -> 距离工具
- 天气随机生成（晴/多云/小雨/中雨）
- 酒店库存随机生成，并覆盖“客满”异常
- 景点随机客流，覆盖“景点人太多”异常
- 内置景点间距离（公里），用于更精确的通勤时间估算
- 最终调用本地 Qwen 生成具体到“上午/下午”的行程安排

## 运行

```bash
cd 05-agent/travel-planning
go run .
```

每次运行都会得到不同场景，便于测试 agent 在正常/异常情况下的安排能力。

如果你使用 Ollama，建议提前拉取模型并设置（可选）：

```bash
ollama pull qwen2.5:7b-instruct
export LLM_BASE_URL=http://127.0.0.1:11434/v1
export LLM_MODEL=qwen2.5:7b-instruct
go run .
```

## 说明

- 本 demo 的目标是“工具调用流程仿真 + LLM 规划能力测试”，不是追求真实 API 数据。
- 如果明日下雨，规划会倾向室内景点（如三星堆博物馆、成都博物馆）。
- 武侯祠、锦里、宽窄巷子之间距离较近，工具数据会体现这一点，便于大模型做动线优化。
